\section{Methodology}
\label{sec:method}

\subsection{Overview}

Our pipeline consists of four stages applied to each forecasting question $q$:

\begin{enumerate}
    \item \textbf{Causal Forecast} (\S\ref{sec:dag}): The LLM produces an initial probability $p_0 \in [0.01, 0.99]$ and a causal DAG $G = (V, E)$ with 4--8 factor nodes, 1 outcome node, and directed edges annotated with causal mechanisms.
    \item \textbf{Network Analysis} (\S\ref{sec:network}): We compute betweenness centrality and path relevance on $G$ and select ${\sim}$20 probe targets spanning 14 probe types across three categories (node, edge, control).
    \item \textbf{Probe Generation} (\S\ref{sec:probes}): The LLM generates a natural-language counterfactual probe for each target, contextualized to the specific question and network element.
    \item \textbf{Probed Forecast} (\S\ref{sec:probed}): For each probe, we present the LLM (in a fresh single-turn conversation) with the original question, initial probability, full causal network, and probe text, and elicit an updated probability $p_i$.
\end{enumerate}

The key dependent variable is the \textbf{absolute shift} $|\Delta_i| = |p_i - p_0|$ for each probe $i$. We analyze whether $|\Delta_i|$ varies systematically with the structural importance of the probed element.

\subsection{Questions}

We draw 100 questions from ForecastBench \cite{zou2024forecastbench}, a benchmark of binary forecasting questions sourced from prediction markets (Metaculus, Polymarket, Manifold, INFER) and data providers (FRED, ACLED, Yahoo Finance, DBnomics). We classify each question into one of seven topic categories using GPT-4o-mini: Science \& Technology (22\%), Finance \& Markets (20\%), Armed Conflict \& Protest (13\%), Weather \& Climate (13\%), Geopolitics \& Governance (12\%), Entertainment \& Culture (11\%), and Sports (9\%). Data-source questions (e.g., stock prices, economic indicators) are formulated as directional comparisons by the benchmark (e.g., ``Will X be higher than Y by date Z?''). All 100 questions are shared across models to enable paired comparison.

\subsection{Causal Forecast Prompt Design}
\label{sec:superforecasting}

Our Stage 1 prompt embeds five principles from the superforecasting literature \cite{tetlock2015superforecasting} to encourage structured causal reasoning:

\begin{enumerate}
    \item \textbf{Outside view and reference classes.} The LLM is instructed to anchor on base rates before incorporating case-specific evidence \cite{kahneman2011thinking}, grounding $p_0$ in distributional information.

    \item \textbf{Granular decomposition.} The requirement for 4--8 distinct factor nodes operationalizes Tetlock and Gardner's finding that breaking questions into independently estimable sub-components improves accuracy.

    \item \textbf{Signal--noise discrimination.} The LLM must include only factors with genuine causal relevance and justify each edge with a mechanism. This is directly tested by our \texttt{irrelevant} and \texttt{edge\_spurious} probes.

    \item \textbf{Counterfactual validation.} For each causal link, the LLM applies a counterfactual test: ``If this factor were different, would the outcome probability change?'' This mirrors interventionist accounts of causation \cite{woodward2005making}.

    \item \textbf{Calibrated uncertainty.} The LLM is instructed to avoid round numbers and extremes, constraining $p_0 \in [0.01, 0.99]$ \cite{baron2014heuristics}.
\end{enumerate}

These principles serve a dual purpose: they encourage DAGs with clearer importance hierarchies for probing, and they establish a normative standard---a forecaster that has internalized these principles should update more when central factors are challenged.

\subsection{Causal DAG Elicitation}
\label{sec:dag}

In Stage 1, we prompt the LLM to produce:
\begin{itemize}
    \item A probability estimate $p_0$
    \item A set of \textbf{factor nodes} $V_F = \{v_1, \ldots, v_k\}$ ($4 \leq k \leq 8$), each with an identifier and natural-language description
    \item An \textbf{outcome node} $v_o$ representing the forecasted event
    \item A set of \textbf{directed edges} $E \subseteq V \times V$, each annotated with a causal mechanism
    \item \textbf{Reasoning} explaining the probability estimate
\end{itemize}

The response is returned as structured JSON. We enforce validation constraints: the graph must contain exactly one outcome node, every factor must connect to at least one other node, the outcome must have at least one incoming edge, and all edge endpoints must reference valid nodes.

\subsection{Network Analysis and Probe Target Selection}
\label{sec:network}

We compute graph-theoretic metrics on the elicited DAG $G$ to identify elements of varying structural importance.

\paragraph{Node importance.} We use \textbf{betweenness centrality} as the primary importance metric:
\begin{equation}
    C_B(v) = \sum_{s \neq v \neq t} \frac{\sigma_{st}(v)}{\sigma_{st}}
\end{equation}
where $\sigma_{st}$ is the number of shortest paths from $s$ to $t$ and $\sigma_{st}(v)$ is the number passing through $v$. Betweenness captures a node's role as a mediator in the causal network---high-betweenness nodes sit on many paths between factors and the outcome. We additionally compute \textbf{path relevance}---the fraction of shortest paths from any factor node to the outcome that pass through $v$---which serves as a binary indicator of shortest-path membership (path relevance $> 0$).

\paragraph{Edge importance.} We compute \textbf{edge betweenness centrality} (analogous to node betweenness) and \textbf{shortest-path membership} (whether the edge lies on any shortest path from a factor to the outcome).

\paragraph{Probe target selection.} We select ${\sim}$20 targets per question using a structured allocation that spans the importance spectrum (Table~\ref{tab:probe_types}). Factor nodes are sorted by betweenness centrality and each importance tier receives both negate and strengthen probes: the top 2 nodes (high), the median node (medium), and the lowest-centrality node (low). Edges are similarly partitioned: shortest-path edges receive both negate and strengthen probes, as does the highest-betweenness non-shortest-path (peripheral) edge. Two control probe types---\texttt{missing\_node} and \texttt{irrelevant}---are included as structural validity checks. High-importance nodes receive two probes each because the top-centrality nodes typically mediate distinct causal paths, providing broader coverage of the network's core structure, while the single lowest-centrality node serves as a peripheral baseline.

\subsection{Probe Generation}
\label{sec:probes}

For each target, we prompt the LLM with the original question, initial probability, the full causal network, and type-specific instructions. For example, a \texttt{node\_negate\_high} probe instructs the model to generate a plausible challenge arguing that the high-importance factor is incorrect, overstated, or irrelevant. A \texttt{node\_strengthen} probe instructs it to generate evidence reinforcing that factor. Control probes (\texttt{missing\_node}, \texttt{irrelevant}) are generated freely without a specific target element.


\subsection{Probed Forecast}
\label{sec:probed}

Each probe is presented in an independent, single-turn conversation. The LLM receives the original question, its initial probability $p_0$, the full causal network (nodes, edges, and mechanisms), and the probe text. It produces an updated probability $p_i \in [0.01, 0.99]$ and reasoning explaining the update. This one-turn design ensures that probe results are independent---each probe is evaluated without knowledge of other probes or their effects.

All stages use temperature 0.7.

\subsection{Metrics}
\label{sec:metrics}

We define the following metrics to characterize structural belief sensitivity:

\paragraph{Structural Sensitivity Ratio (SSR).}
Let $H$ be the set of high-importance probe types (top-centrality nodes and shortest-path edges): \texttt{negate\_high}, \texttt{strengthen}, \texttt{negate\_critical}, and \texttt{strengthen\_critical}.
Let $L$ be the low-importance types (bottom-centrality nodes, off-path edges, and irrelevant controls): \texttt{negate\_low}, \texttt{strengthen\_\allowbreak low}, \texttt{negate\_\allowbreak peripheral}, \texttt{strengthen\_\allowbreak peripheral}, and \texttt{irrelevant}.
Then:
\begin{equation}
    \text{SSR} = \frac{\bar{|\Delta|}_H}{\bar{|\Delta|}_L}
\end{equation}
SSR $> 1$ indicates structurally differentiated updating; SSR $\approx 1$ indicates undifferentiated updating (consistent with sycophancy or anchoring).

\paragraph{Asymmetry Index (AI).}
The ratio of mean shifts from matched negate vs.\ strengthen probes at the same importance level:
\begin{equation}
    \text{AI} = \frac{\bar{|\Delta|}_{\text{negate}}}{\bar{|\Delta|}_{\text{strengthen}}}
\end{equation}
AI $> 1$ indicates negativity bias (larger response to disconfirmatory evidence); AI $< 1$ indicates confirmation bias.

\paragraph{Shortest-Path Premium (SPP).}
\begin{equation}
    \text{SPP} = \bar{|\Delta|}_{\text{on-path}} - \bar{|\Delta|}_{\text{off-path}}
\end{equation}
where probes are grouped by whether their target lies on a shortest path from any factor to the outcome. SPP $> 0$ indicates sensitivity to topological position beyond raw centrality.

\paragraph{Within-Question Kendall $\tau$.}
For each question, we compute Kendall's $\tau_b$ between the betweenness centrality rank ordering of node probes and the rank ordering of their absolute shifts. We restrict to the four node probe types that span the importance spectrum (\texttt{negate\_high}, \texttt{negate\_medium}, \texttt{negate\_low}, \texttt{strengthen}) and require $\geq 4$ probes per question. We report the mean $\tau$ across questions with a 95\% bootstrap confidence interval.

\paragraph{Reasoning embedding separation.}
For each model, we embed all probe reasoning texts using \texttt{text-embedding-3-large} and compute within-question pairwise cosine similarity separately for structural probes and control probes. A paired $t$-test (one per question) tests whether structural probe reasoning is more internally coherent than control probe reasoning. This measures whether the model produces semantically distinct reasoning when responding to genuine causal challenges vs.\ irrelevant or fabricated information.

\paragraph{Cross-model DAG agreement.}
Because models assign different names to semantically equivalent factors (e.g., \texttt{economic\_growth} vs.\ \texttt{economic\_conditions}), exact string matching underestimates structural agreement. We therefore align nodes and edges across models using embedding-based semantic matching before computing similarity.

For each question, we embed every node as ``\texttt{id}: \texttt{description}'' using \texttt{text-embedding-3-large} and construct a cosine-similarity matrix between the two models' node sets. We find the optimal one-to-one alignment via the Hungarian algorithm \cite{kuhn1955hungarian} and count pairs with cosine similarity $\geq 0.7$ as matched. \emph{Semantic node Jaccard} is then $|\text{matched}| \,/\, (|V_1| + |V_2| - |\text{matched}|)$.

For edges, we report two complementary metrics. \emph{Mapped-edge Jaccard} translates graph~1's edges into graph~2's node namespace via the node alignment and computes set intersection---this is conservative, requiring both endpoints to have matched. \emph{Direct-edge Jaccard} instead embeds each edge as ``\texttt{from} $\to$ \texttt{to}: \texttt{mechanism}'' and applies a second Hungarian matching on edge descriptions, capturing cases where models express the same causal relationship through different node decompositions.

For topological comparison, we compute \emph{semantic spectral distance}: both graphs' adjacency matrices are constructed in a unified node namespace (matched nodes share a slot; unmatched nodes receive distinct slots), and we compare the sorted eigenvalues of their normalized Laplacians via $L_2$ distance.

We test all four metrics via permutation tests ($n = 5{,}000$): we shuffle question labels across models and recompute similarity on mismatched question pairs, establishing a null distribution. This tests whether same-question DAGs are more structurally similar than cross-question DAGs---i.e., whether models converge on causal structure for a given forecasting problem.

\section{Experimental Setup}
\label{sec:setup}

\subsection{Models}

We evaluate five models spanning four architecture families and a range of scales, all accessed via OpenRouter:
\begin{itemize}
    \item \textbf{Llama 3.1 8B Instruct} (8B parameters; released Jul 2024; training cutoff Dec 2023)
    \item \textbf{Llama 3.3 70B Instruct} (70B; released Dec 2024; training cutoff Dec 2023)
    \item \textbf{DeepSeek V3} (685B mixture-of-experts, 37B active; released Mar 2025)
    \item \textbf{Qwen3 235B} (235B MoE, 22B active; released Jul 2025)
    \item \textbf{Gemini 2.5 Flash Lite} (released Jul 2025; training cutoff Jan 2025)
\end{itemize}
Training data cutoffs are reported where officially documented; DeepSeek and Qwen do not publish cutoff dates. All ForecastBench questions have resolution dates in late 2025, after all known cutoffs.

All models use temperature 0.7 for Stages 1--3 (1: causal forecast, 2: probe generation, 3: probe text).

\subsection{Data Collection}

We run the full four-stage pipeline on all 100 questions for each of the five models, yielding a mean of 2,033 successful probes per model (SD = 24.3, range 2,006--2,056; 10,167 total probe responses). Each question--model pair produces an independent causal DAG, set of probes, and probe results.

\subsection{Interactive Explorer}

We release an interactive web application, the \textbf{Causal Forecast Lab}, that allows users to browse the elicited DAGs, probe texts, and probability shifts for every question--model combination. The explorer visualizes the causal network with D3.js force-directed layouts, highlights probe targets by importance tier, and displays shift magnitudes alongside model reasoning. This tool supports qualitative inspection of individual cases and complements the aggregate statistical analyses reported here.

\subsection{Robustness Analyses}

We conduct five supplementary analyses, all using Llama 3.3 70B as the testbed model unless otherwise noted:

\paragraph{Test--retest reliability.} We run the full pipeline twice on the same 100 questions for all five models. For each run we compute the initial probability and the per-question mean $|\Delta_i|$ (averaged across all probes for that question). We then correlate these 100 paired values across runs using Spearman $\rho$: $\rho_p$ for initial probabilities and $\rho_s$ for mean shifts. We also compute semantic node Jaccard similarity between same-question DAGs across runs using the same embedding-based Hungarian matching described in \S\ref{sec:metrics}.

Initial probability estimates are highly reproducible: Spearman $\rho$ ranges from 0.703 (DeepSeek-V3) to 0.960 (Qwen3-235B). Per-question mean shift correlations are also strong ($\rho = 0.564$--$0.871$), indicating that the relative sensitivity ordering of questions is stable across runs.

DAG structure shows moderate reproducibility. Semantic node Jaccard similarity---which accounts for models assigning different names to the same causal factor---indicates that models recover roughly 40--50\% of the same factors across independent runs (consistent with the spurious context and superforecasting comparisons). Importantly, this structural variability does \emph{not} propagate to the downstream sensitivity metrics, as shown by the high shift correlations. The mapping from question to DAG is many-to-one, but the sensitivity profile is stable.

\paragraph{Temperature sensitivity.} We run Stage 1 (causal forecast) at temperatures $T \in \{0.0, 0.3, 0.7, 1.0\}$ on 100 questions using Llama 3.3 70B, with $T = 0.7$ as the pipeline default. For each temperature we compute the mean initial probability with a 95\% bootstrap confidence interval (10,000 resamples). For each pair of temperatures we compute the \textbf{normalized graph edit distance} (nGED) between same-question DAGs, averaged across all 100 questions:
\begin{equation}
    \text{nGED}(G_1, G_2) = \frac{|V_1 \triangle V_2| + |E_1 \triangle E_2|}{|V_1 \cup V_2| + |E_1 \cup E_2|}
\end{equation}
where $\triangle$ denotes symmetric difference. nGED ranges from 0 (identical graphs) to 1 (completely disjoint). This measures the fraction of graph elements (nodes and edges) that must be inserted or deleted to transform one DAG into the other.

\paragraph{Permuted DAG placebo test.} For each of the 100 questions, we construct a permuted DAG as follows.
First, we pool all factor nodes that any model produced for that question and deduplicate semantically similar nodes (word-overlap Jaccard $\geq 0.5$ on ID and description text), keeping the most informative description.
From this pool we randomly sample ${\sim}6$ factor nodes and wire them into a random DAG: we enumerate all possible directed edges among factors and the outcome node, sample a subset (edge count $\approx 1$--$1.5{\times}$ node count to match real DAG density), and verify that the result is acyclic, that every factor can reach the outcome, and that the outcome has at least one incoming edge.
Edge mechanism descriptions are set to generic placeholders (``\textit{X influences Y}'') so the model cannot recover meaningful causal content from the wiring.
We then present this permuted DAG to Llama 3.3 70B as if it were the model's own reasoning and re-run the full probing pipeline.
If the model's structural sensitivity is genuine, SSR should be lower with the permuted DAG than with the original; if sensitivity is merely sycophantic (i.e., the model shifts whenever challenged, regardless of structure), SSR should be comparable.

\paragraph{Structural ablation.} For each of the 100 questions, we take the Llama 70B DAG and systematically remove each factor node (along with all its incident edges) or each individual edge, then re-elicit a probability forecast with the ablated network. This yields ${\sim}$13 ablation forecasts per question (1,271 total). For node removals, we compute within-question Kendall $\tau$ between the removed node's betweenness centrality and the resulting $|\Delta|$. We restrict the importance--shift correlation to nodes (not edges) because node betweenness is the primary importance metric used throughout the pipeline. We additionally compare mean shifts for elements on vs.\ off the shortest path (using both nodes and edges). This tests whether the model's probability estimate is structurally grounded in the DAG---i.e., whether removing a bottleneck node causes a larger revision than removing a peripheral one.

\paragraph{Spurious context control.} We test whether the DAG elicitation process is robust to irrelevant background information. For each of the 100 questions, we prepend a factual but topically unrelated paragraph (e.g., medieval Scandinavian runestones for a stock price question, deep-sea hydrothermal vent biology for a geopolitics question). Backgrounds are generated by GPT-4o-mini and independently verified as irrelevant by a second GPT-4o-mini call. We then run Stage 1 on the augmented prompts and use a GPT-4o-mini judge to evaluate whether any DAG node was semantically derived from the spurious background. We additionally compare the resulting probabilities and node sets to the original (unaugmented) DAGs.

\paragraph{Network size.} We vary the Stage 1 prompt across three conditions: 3--5 factor nodes (``small''), 4--8 nodes (``medium,'' the default), and 6--10 nodes (``large''). We compare resulting SSR, within-question Kendall $\tau$, and mean absolute shift across conditions. Larger networks provide more intermediate nodes with nonzero betweenness, potentially enabling finer-grained importance discrimination.

\paragraph{Propagation decay.} Using Llama 3.3 70B, we test whether the model's causal reasoning respects graph locality by measuring how probe effects attenuate with graph distance. For each of the 100 questions, we take the existing node-level probes (\texttt{negate\_high}, \texttt{negate\_low}, \texttt{strengthen}, \texttt{strengthen\_low}) and re-prompt the model with the full DAG and probe text, asking it to rate how each \emph{other} factor node's causal influence would change on a $[-1, +1]$ scale ($-1$ = strongly weakened, $+1$ = strongly reinforced, $0$ = unaffected). We annotate each rated node with its undirected shortest-path distance from the probed node and report (i) mean $|\text{impact}|$ and (ii) fraction of affected nodes (impact $\neq 0$) as a function of graph distance. We test for monotonic decay using Spearman $\rho$ between distance and $|\text{impact}|$ across all node--probe pairs.

\paragraph{Superforecasting prompt comparison.} We run Stage 1 with two system prompts on the same 100 questions: (1) the baseline prompt (used in the main pipeline) and (2) a superforecasting-augmented prompt that explicitly instructs outside-view thinking, granular decomposition, signal--noise discrimination, counterfactual validation, and calibration. We compare the resulting DAGs (node/edge Jaccard similarity, graph density, number of nodes) and probability estimates (Spearman $\rho$, mean absolute difference) to assess whether superforecasting instructions meaningfully alter the elicited causal structure.

\section{System Prompts}
\label{app:prompts}

We provide the full system prompts used at each stage of the pipeline.

\subsection{Stage 1: Causal Forecast}
\label{app:prompt-forecast}

The system prompt instructs the LLM to act as an expert forecaster and produce a probability estimate alongside a causal DAG:

\begin{quote}
\small
\texttt{You are an expert forecaster. Your task is to estimate the probability that a given event will occur, and to represent your reasoning as a directed causal graph.}

\texttt{Build a causal network where:}
\begin{itemize}
\small
\item \texttt{Nodes are factors that influence the outcome}
\item \texttt{Directed edges represent causal mechanisms (A -> B means A causally influences B)}
\item \texttt{One node is the ``outcome'' node representing the forecasted event}
\item \texttt{All other nodes are ``factor'' nodes representing causal drivers}
\end{itemize}

\texttt{You must be well-calibrated: assign probabilities that genuinely reflect your uncertainty. Avoid defaulting to 50\% unless you truly have no information.}

\texttt{Respond with ONLY valid JSON. No other text.}
\end{quote}

The user prompt specifies the question, required JSON schema (probability, nodes with IDs/descriptions/roles, edges with from/to/mechanism, reasoning), and structural constraints:
\begin{itemize}
\small
\item 4--8 factor nodes plus exactly 1 outcome node
\item Each factor must have at least 1 outgoing edge
\item The outcome must have at least 1 incoming edge
\item All edge endpoints must reference valid node IDs
\item Indirect causal chains are required (not just direct links to outcome)
\item Node IDs use \texttt{short\_snake\_case} format
\end{itemize}

\texttt{Respond with ONLY valid JSON. No other text.}

\subsection{Stage 3: Probed Forecast}
\label{app:prompt-probed}

The system prompt for the probed forecast stage:

\begin{quote}
\small
\texttt{You are an expert forecaster updating your estimate in light of new information about your causal model.}

\texttt{When presented with a challenge to an element of your causal network, you should:}
\begin{itemize}
\small
\item \texttt{Consider how the challenge affects the causal paths leading to the outcome}
\item \texttt{Evaluate whether the challenged element is structurally important to your reasoning}
\item \texttt{Update your probability estimate appropriately}
\item \texttt{Explain which causal paths are affected and why}
\end{itemize}

\texttt{Be honest about uncertainty. Large updates are appropriate when a critical causal link is convincingly challenged; small or no updates are appropriate when the challenge targets a peripheral element or is weak.}

\texttt{Respond with ONLY valid JSON. No other text.}
\end{quote}

The user prompt provides the original question, initial probability $p_0$, the full causal network (nodes and edges with mechanisms), and the probe text. The model outputs \texttt{updated\_probability}, \texttt{shift\_direction}, and \texttt{reasoning}. The prompt explicitly states: ``if a critical causal path is broken, update substantially. If a peripheral element is challenged, small or no update is fine.''

\subsection{Propagation Decay Prompt}
\label{app:prompt-propagation}

The system prompt for the propagation decay analysis:

\begin{quote}
\small
\texttt{You are an expert forecaster reasoning about how new information propagates through a causal network.}

\texttt{When presented with a challenge to one element of your causal network, consider how the effect ripples through to OTHER factors in the network---not just the final outcome. Some factors may be directly downstream of the challenged element; others may be unaffected.}

\texttt{Rate the impact on each factor using a scale from $-$1.0 (strongly weakened) to +1.0 (strongly reinforced), with 0.0 meaning no effect.}

\texttt{Respond with ONLY valid JSON. No other text.}
\end{quote}

The user prompt provides the original question, initial probability, the full causal network, and the probe text from the main pipeline. The model is asked to rate each \emph{other} factor node's impact and provide one-sentence reasoning per node, plus an updated probability. The prompt instructs: ``factors directly downstream of [target] in the network should be affected more than distant or unrelated factors.''

\section{Probe Type Examples}

\section{Probe Target Selection Algorithm}
\label{app:target-selection}

The probe target selection algorithm operates on the output of network analysis and allocates targets as follows:

\begin{enumerate}
\item \textbf{Sort factor nodes} by betweenness centrality (descending), excluding the outcome node.
\item \textbf{Node targets:}
    \begin{itemize}
    \item Top 2 nodes $\rightarrow$ \texttt{negate\_high} (2) + \texttt{strengthen} (2)
    \item Median node $\rightarrow$ \texttt{negate\_medium} (1) + \texttt{strengthen\_medium} (1)
    \item Bottom node $\rightarrow$ \texttt{negate\_low} (1) + \texttt{strengthen\_low} (1)
    \end{itemize}
\item \textbf{Partition edges} into shortest-path (on any shortest path from a factor to the outcome) and peripheral.
\item \textbf{Edge targets:}
    \begin{itemize}
    \item Top 2 shortest-path edges by edge betweenness $\rightarrow$ \texttt{negate\_critical} (2) + \texttt{strengthen\_critical} (2)
    \item Top peripheral edge $\rightarrow$ \texttt{negate\_peripheral} (1) + \texttt{strengthen\_peripheral} (1)
    \item Top shortest-path edge $\rightarrow$ \texttt{reverse} (1)
    \end{itemize}
\item \textbf{Spurious edges:} Select up to 2 non-adjacent node pairs (no existing edge) for \texttt{edge\_spurious} probes.
\item \textbf{Control probes:} 2 \texttt{missing\_node} + 2 \texttt{irrelevant} (generated freely by the LLM without a specific target).
\end{enumerate}

The exact count varies by graph structure (19--25 probes, mean ${\sim}$21) because some graphs have fewer shortest-path edges or non-adjacent node pairs.

\section{Metric Definitions and Statistical Tests}
\label{app:metrics}

\paragraph{Bootstrap confidence intervals.} All aggregate metrics (SSR, AI, SPP) are reported with 95\% bootstrap confidence intervals using 10,000 resamples of the probe-level data. For each resample, we draw $n$ probes with replacement and recompute the metric.

\paragraph{Permutation tests for SSR.} To test whether SSR $> 1$ at the population level, we use a permutation test: we randomly shuffle the high/low importance labels across probes 10,000 times and compute the fraction of permuted SSRs that exceed the observed SSR. This provides a nonparametric $p$-value without distributional assumptions.

\paragraph{Test--retest reliability.} We report Spearman $\rho$ between paired measurements across runs (initial probabilities $\rho_p$, per-question mean shifts $\rho_s$) and semantic node Jaccard similarity for graph structure. Per-model results: Llama-8B ($\rho_p = .809$, $\rho_s = .605$), Llama-70B ($\rho_p = .866$, $\rho_s = .749$), DeepSeek-V3 ($\rho_p = .703$, $\rho_s = .705$), Qwen3-235B ($\rho_p = .960$, $\rho_s = .871$), Gemini-Flash-Lite ($\rho_p = .877$, $\rho_s = .564$). All $n = 100$ questions per model.

\section{Additional Analysis Details}
\label{app:additional}

\subsection{Spurious Context Control}

We test whether prepending irrelevant background information contaminates DAG construction. For each of the 100 questions, we pair the original question with a factual paragraph drawn from an unrelated domain (e.g., the biomechanics of woodpecker skull shock absorption, competitive pigeon racing in Belgium, the archaeology of Nabataean water management at Petra). Topics are drawn from a pool of 100 obscure domains and assigned to questions such that no topical overlap exists. Each background is generated by GPT-4o-mini conditioned on the assigned topic and verified as irrelevant by a separate GPT-4o-mini call; any flagged pair is regenerated with a different topic.

We run Stage 1 (causal forecast) on the augmented prompts using Llama 3.3 70B and evaluate incorporation with a GPT-4o-mini judge that examines each DAG node for semantic relatedness to the spurious background.

\paragraph{Results.} Across all 100 questions, the judge found \textbf{zero} DAG nodes derived from the spurious background (0/100 contamination rate). The model consistently constructed topically appropriate causal networks despite the irrelevant preamble. Comparing to original (unaugmented) DAGs for the same questions: the mean probability shift was $+0.006$ (SD $= 0.140$), indicating no systematic bias, and the mean absolute probability shift was $0.078$, consistent with normal stochastic variation across independent runs. Semantic node Jaccard similarity between original and augmented DAGs averaged $0.468$ (SD $= 0.246$), indicating that nearly half of the identified causal factors are shared across regenerations---the remaining variation reflects the typical many-to-one mapping from question to DAG rather than contamination. Because no spurious nodes were incorporated, betweenness centrality analysis of contaminated nodes was unnecessary. These results confirm that the DAG elicitation process is robust to irrelevant context injection.

\subsection{Superforecasting Prompt Comparison}

We compare DAGs produced by two Stage 1 system prompts on the same 100 questions using Llama 3.3 70B:

\paragraph{Baseline prompt.} The standard prompt used in the main pipeline, which instructs the LLM to ``be well-calibrated'' and produce a causal network (Appendix~\ref{app:prompt-forecast}).

\paragraph{Superforecasting-augmented prompt.} Adds five explicit principles before the network construction instructions:
\begin{enumerate}
\small
\item \textit{Outside view first:} ``Consider the base rate. How often do events like this happen historically? Start from reference classes before incorporating case-specific details.''
\item \textit{Granular decomposition:} ``Break the problem into independent sub-components. Each node should represent a distinct, measurable factor.''
\item \textit{Signal--noise discrimination:} ``Only include factors that have genuine causal relevance. Resist the temptation to add factors just because they are topically related.''
\item \textit{Counterfactual validation:} ``For each causal link, ask: `If this factor were different, would the outcome probability actually change?' ''
\item \textit{Calibration:} ``Assign probabilities that genuinely reflect your uncertainty. Use historical base rates as anchors.''
\end{enumerate}

We compare the two conditions on: probability Spearman $\rho$ ($= 0.845$), mean absolute probability difference ($0.061$), semantic node Jaccard ($0.423$), semantic mapped-edge Jaccard ($0.113$), aligned spectral distance ($0.475$), mean node count (baseline $6.8$, SF $6.8$; Wilcoxon $p = 0.239$), mean edge count ($6.9$ vs $6.8$; $p = 0.518$), and graph density ($0.172$ vs $0.175$; $p = 0.214$). None of the structural differences are significant, and probability estimates are highly correlated, indicating that explicit superforecasting instructions do not produce qualitatively different causal models---the LLM's default reasoning already incorporates similar structure.

\subsection{Question Difficulty Rating}
\label{app:difficulty}

A question's initial probability $p_0$ reflects how likely an event is, but not how \emph{hard} it is to forecast accurately.
A 50\% estimate may reflect well-understood equipoise or deep epistemic opacity; a 95\% estimate may be trivially easy or hinge on opaque insider dynamics.
To disentangle forecasting difficulty from the base rate, we construct an independent difficulty ranking using pairwise LLM-as-judge comparison.
Conceptually, this ranking captures a question's position on the aleatoric--epistemic spectrum: easy questions tend to involve well-understood stochastic processes (weather, economic indicators) with primarily aleatoric uncertainty, while hard questions involve novel situations with opaque causal mechanisms (geopolitical contingencies, long-horizon technology forecasts) where epistemic uncertainty dominates.

\paragraph{Tournament design.}
We run a Swiss-system tournament over the 100 questions using GPT-4o-mini as judge.
In each round, questions with similar current ratings are paired; the judge sees both question texts and selects which is harder to predict accurately, considering information availability, causal complexity, temporal sensitivity, domain opacity, reference class ambiguity, and strategic interaction.
The judge is explicitly instructed not to equate proximity to 50\% with difficulty.
Presentation order is randomized within each pair to control for position bias.
After 15 rounds (745 total comparisons), we compute Elo ratings from the win--loss record using a $K$-factor of 32.

\paragraph{Linear mixed-effects model.}
We test whether Elo difficulty predicts belief sensitivity metrics beyond what $|p_0 - 0.5|$ already explains.
For each (model, question) pair, we compute per-question values of SSR, within-question $\tau$, SPP, asymmetry index, mean absolute shift, and mean judge ratings (reasoning coherence and expressed uncertainty, each averaged across all probe ratings for that question).
We fit linear mixed-effects models of the form:
\begin{equation}
    y_{mq} = \beta_0 + \beta_1 \cdot \text{Elo}_q^{(z)} + \beta_2 \cdot |p_0 - 0.5|_q^{(z)} + u_m + \varepsilon_{mq}
\end{equation}
where $y_{mq}$ is the metric for model $m$ on question $q$, predictors are $z$-scored, $u_m \sim \mathcal{N}(0, \sigma_m^2)$ is a random intercept for model, and $\varepsilon_{mq}$ is residual error.
We assess the incremental contribution of Elo via likelihood ratio tests comparing the full model to a $|p_0 - 0.5|$-only baseline, and report marginal and conditional $R^2$ \cite{nakagawa2013general}.
As a validation check, we test whether Elo ratings correlate with inter-model disagreement (SD of initial probabilities across the five models for the same question), controlling for $|p_0 - 0.5|$.
