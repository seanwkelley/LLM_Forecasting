STUDY DESIGN
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

RESEARCH QUESTIONS:

1. Can humans + AI assistants forecast geopolitical outcomes accurately
   using only publicly available information?

2. How does Human+AI (public info) compare to LLM aggregator (full info)?

3. Do AI assistants improve human forecasting accuracy and calibration?

4. Can forecasters distinguish genuine patterns (TRUE) from noise (CONTROL)?

EXPERIMENTAL CONDITIONS:

TRUE CONDITION:
  • Real simulation dynamics
  • Information causally connected to outcomes
  • Tests forecasting accuracy on genuine signal

CONTROL CONDITION:
  • Randomized information
  • Information-outcome connection broken
  • Tests robustness to overfitting
  • Good forecasters should cluster around base rate (~35%) on CONTROL

PROCEDURE:

1. Recruit participants (N = 20-30 recommended)
   • Professional forecasters (ideal)
   • Graduate students in IR/political science
   • Intelligence analysts
   • Interested volunteers with analytical background

2. Assign to condition (within-subject or between-subject)
   • WITHIN: Each participant does BOTH TRUE and CONTROL
   • BETWEEN: Half get TRUE, half get CONTROL

3. Pair with AI assistant
   • Recommended: Claude (Sonnet/Opus), GPT-4, Gemini Pro
   • Participants share briefings with AI and discuss
   • AI helps analyze but does NOT make final forecast

4. Collect forecasts
   • 10 periods per condition
   • Probability (0-100%), confidence, rationale
   • Record in human_forecasts_template.csv

5. Analyze performance
   • Accuracy: Brier score, log score
   • Calibration: Are 70% forecasts correct ~70% of time?
   • Discrimination: Can they distinguish high/low probability events?
   • Control performance: Do forecasts cluster around base rate?

TROUBLESHOOTING
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

ISSUE: Participants confused about what counts as "collapse"
  → Point them to definition in PREDICTION_CHALLENGE_INSTRUCTIONS.txt
  → Emphasize: government REMOVED from power, not just weakened

ISSUE: Participants taking too long per period (>30 minutes)
  → Encourage them to use AI assistant more efficiently
  → Remind them 15-20 minutes is target, not requirement
  → Consider if instructions need simplification

ISSUE: AI assistants giving direct probability estimates
  → Instruct participants to ask AI for analysis, not predictions
  → Prompt the AI: "Help me analyze but don't give me a number"

ISSUE: Participants requesting more information
  → Explain realistic information constraint is intentional
  → Professional forecasters face same limitations
  → Cannot provide internal deliberations (would contaminate study)

ISSUE: Forecasts all cluster around 50% (uninformative)
  → Encourage stronger evidence weighting
  → Review forecasting principles in instructions
  → May indicate participant disengagement (consider exclusion)

ISSUE: Wild swings period-to-period (e.g., 20% → 80% → 30%)
  → Encourage incremental updating principle
  → Review "Update Incrementally" in instructions
  → May indicate lack of forecasting experience

REGENERATING PROMPTS:
  If you need to regenerate with different simulation:

  ```R
  source("src/forecast_prompts_public.R")
  state <- readRDS("outputs/simulation_state.rds")

  # Generate TRUE condition
  generate_all_public_forecast_prompts(
    state,
    "outputs/human_forecasting/forecasting_prompts_TRUE.txt",
    control_condition = FALSE
  )

  # Generate CONTROL condition
  generate_all_public_forecast_prompts(
    state,
    "outputs/human_forecasting/forecasting_prompts_CONTROL.txt",
    control_condition = TRUE
  )
  ```

DATA STRUCTURE:
  • state$state_history[[period]]$scenario_state - Public metrics per period
  • state$events_history[[period]] - External events per period
  • state$action_results[[period]] - Actions taken per period
  • state$assessments_history[[period]] - LLM forecasts per period
