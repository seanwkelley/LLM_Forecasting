\section{Methodology}
\label{sec:method}

\subsection{Overview}

Our pipeline consists of four stages applied to each forecasting question $q$:

\begin{enumerate}
    \item \textbf{Causal Forecast} (\S\ref{sec:dag}): The LLM produces an initial probability $p_0 \in [0.01, 0.99]$ and a causal DAG $G = (V, E)$ with $k$ factor nodes (default $6 \leq k \leq 10$; $12 \leq k \leq 16$ in the network size condition), 1 outcome node, and directed edges annotated with causal mechanisms.
    \item \textbf{Network Analysis} (\S\ref{sec:network}): We compute betweenness centrality and outcome mediation on $G$ and select ${\sim}$21 probe targets spanning 14 probe types across four categories (strengthen, negate, structural challenge, control).
    \item \textbf{Probe Generation} (\S\ref{sec:probes}): The LLM generates a natural-language counterfactual probe for each target, contextualized to the specific question and network element.
    \item \textbf{Probed Forecast} (\S\ref{sec:probed}): For each probe, we present the LLM (in a fresh single-turn conversation) with the original question, initial probability, full causal network, and probe text, and elicit an updated probability $p_i$.
\end{enumerate}

The key dependent variable is the magnitude of probability revision for each probe $i$. Because the model generates its own DAG in Stage~1, we are not testing whether LLMs perform causal reasoning \emph{de novo}---we are testing whether their probability updates are \textbf{internally consistent} with their own elicited causal structure. A model that places a factor on many paths to the outcome has implicitly committed to that factor's importance; we test whether it honors that commitment when the factor is challenged.

\subsection{Questions}

We draw questions from ForecastBench \cite{zou2024forecastbench}, a benchmark of 500 binary forecasting questions sourced from prediction markets (Metaculus, Polymarket, Manifold, INFER) and data providers (FRED, ACLED, Yahoo Finance, DBnomics). All questions have resolution dates in late 2025, after the training data cutoffs of all evaluated models.

\paragraph{Causal complexity filtering.} Because our pipeline elicits causal DAGs with 6--16 factor nodes, we require questions whose outcomes are shaped by a genuine network of interacting causal factors across multiple domains. We classify all 500 ForecastBench questions using GPT-5.4 (\texttt{openai/gpt-5.4}) with a structured prompt that applies five criteria: (1)~a domain expert would naturally identify $12+$ distinct, non-redundant causal factors that \emph{interact} with each other; (2)~these factors span $3+$ domains (political, economic, social, technological, military, regulatory, environmental, organizational); (3)~the factors form a genuine causal \emph{network} with indirect pathways and mediating variables, not a linear chain; (4)~the question is \emph{grounded} in real, identifiable actors or measurable conditions with current observable evidence; and (5)~the natural expert reasoning involves understanding how factors interact, not simpler approaches such as time-series extrapolation, decision-tree analysis, tournament/competition modeling, pipeline/stage-gate reasoning, or market pricing.

The key distinction is between questions whose drivers form an interacting network (e.g., geopolitical conflicts where military, diplomatic, economic, and domestic factors feed back on each other) versus questions better answered by simpler methods (e.g., week-over-week stock price movements, single-domain award outcomes, speculative scenarios with no historical precedent). Questions that resolve as a single discrete event are classified as HIGH if the \emph{conditions} that determine that event emerge from a complex multi-domain network.

This yields 116 high-complexity questions (23.2\% of the pool). These span geopolitical conflicts, multi-stakeholder policy decisions, technology ecosystem dynamics, conflict escalation scenarios, and cross-domain regulatory questions. The 384 excluded questions are predominantly time-series extrapolations (stock prices, weather comparisons, financial indicators), single-domain outcomes (sports, entertainment awards), speculative/ungrounded scenarios, and narrow institutional decisions. All 116 questions are shared across models to enable paired comparison.

\subsection{Causal Forecast Prompt Design}
\label{sec:superforecasting}

Our Stage 1 prompt is deliberately minimal: the LLM is instructed to act as an ``expert forecaster,'' estimate a probability, and represent its reasoning as a directed causal graph (see Appendix~\ref{app:prompt-forecast} for the full prompt). The prompt specifies structural requirements---6--10 factor nodes, one outcome node, directed edges with causal mechanisms, indirect chains not just direct links---but does not instruct the model on \emph{how} to reason. There are no instructions about base rates, decomposition strategies, counterfactual testing, or calibration heuristics. This design ensures that the elicited causal structure reflects the model's own reasoning rather than experimenter-imposed scaffolding.

\subsection{Causal DAG Elicitation}
\label{sec:dag}

In Stage 1, we prompt the LLM to produce:
\begin{itemize}
    \item A probability estimate $p_0$
    \item A set of \textbf{factor nodes} $V_F = \{v_1, \ldots, v_k\}$ ($6 \leq k \leq 10$), each with an identifier and natural-language description
    \item An \textbf{outcome node} $v_o$ representing the forecasted event
    \item A set of \textbf{directed edges} $E \subseteq V \times V$, each annotated with a causal mechanism
    \item \textbf{Reasoning} explaining the probability estimate
\end{itemize}

The response is returned as structured JSON. We enforce validation constraints: the graph must contain exactly one outcome node, every factor must connect to at least one other node, the outcome must have at least one incoming edge, and all edge endpoints must reference valid nodes.

\subsection{Network Analysis and Probe Target Selection}
\label{sec:network}

We compute graph-theoretic metrics on the elicited DAG $G$ to identify elements of varying structural importance.

\paragraph{Node importance.} We use \textbf{betweenness centrality} as the primary importance metric:
\begin{equation}
    C_B(v) = \sum_{s \neq v \neq t} \frac{\sigma_{st}(v)}{\sigma_{st}}
\end{equation}
where $\sigma_{st}$ is the number of shortest paths from $s$ to $t$ and $\sigma_{st}(v)$ is the number passing through $v$. Betweenness captures a node's role as a mediator in the causal network---high-betweenness nodes sit on many paths between factors and the outcome. We additionally compute \textbf{outcome mediation}---the fraction of shortest paths from any factor node to the outcome that pass through $v$---which captures a node's role as a mediator specifically on outcome-directed paths. Unlike betweenness (which counts paths between all pairs), outcome mediation is outcome-directed and serves as a continuous measure of how central the node is to the forecasting target.

\paragraph{Edge classification.} For probe target selection, edges are classified by \textbf{shortest-path membership} (whether the edge lies on any shortest path from a factor to the outcome). Shortest-path edges receive both negate and strengthen probes; non-shortest-path (peripheral) edges serve as edge-level baselines.

\paragraph{Probe target selection.} We select ${\sim}$20 targets per question using a structured allocation that spans the importance spectrum (Table~\ref{tab:probe_types}). Factor nodes are sorted by betweenness centrality and each importance tier receives both negate and strengthen probes: the top 2 nodes (high), the median node (medium), and the lowest-centrality node (low). Edges are similarly partitioned: shortest-path edges receive both negate and strengthen probes, as does a non-shortest-path (peripheral) edge. Two control probe types---\texttt{missing\_node} and \texttt{irrelevant}---are included as structural validity checks. High-importance nodes receive two probes each because the top-centrality nodes typically mediate distinct causal paths, providing broader coverage of the network's core structure, while the single lowest-centrality node serves as a peripheral baseline.

\subsection{Probe Generation}
\label{sec:probes}

For each target, we prompt the LLM with the original question, initial probability, the full causal network, and type-specific instructions (Table~\ref{tab:probe_instructions}). For example, a node negate probe instructs the model to generate a plausible challenge arguing that the target factor is incorrect or irrelevant; a node strengthen probe instructs it to generate evidence reinforcing that factor. Importantly, the probe generation instructions do not reveal the structural importance of the target---the same instruction template is used for high-, medium-, and low-importance nodes, ensuring that any linguistic differences between probes reflect the semantic content of the target rather than experimenter-imposed framing. Structural challenge probes (\texttt{edge\_reverse}, \texttt{edge\_structural}, \texttt{missing\_node}) question the graph architecture rather than variable states. Control probes (\texttt{irrelevant}) are generated freely without a specific structural target.


\subsection{Probed Forecast}
\label{sec:probed}

Each probe is presented in an independent, single-turn conversation. The LLM receives the original question, its initial probability $p_0$, the full causal network (nodes, edges, and mechanisms), and the probe text. It produces an updated probability $p_i \in [0.01, 0.99]$ and reasoning explaining the update. This one-turn design ensures that probe results are independent---each probe is evaluated without knowledge of other probes or their effects.

All stages use temperature 0.7.

\subsection{Statistical Framework}
\label{sec:metrics}

Our primary analysis uses \textbf{linear mixed-effects (LME) regression} (fitted via \texttt{lme4::lmer} in R with Satterthwaite $p$-values from \texttt{lmerTest}) to test whether topological properties of the probed element predict the magnitude of probability revision. This approach uses the full variance of continuous topological predictors while accounting for question-level clustering and model-specific baselines.

\paragraph{Dependent variable.}
Because raw probability shifts are mechanically bounded (a forecast of $p_0 = 0.90$ can shift at most $+0.09$ upward but $-0.89$ downward), we use the \textbf{absolute log-odds shift} as our primary dependent variable:
\begin{equation}
    |\Delta\text{logit}|_i = \left| \log\frac{p_i}{1-p_i} - \log\frac{p_0}{1-p_0} \right|
\end{equation}
This transformation removes boundary compression and places shifts on an unbounded scale. We additionally include \textbf{probe direction} (negate vs.\ strengthen) as a fixed-effect covariate to control for asymmetric updating---models may systematically shift more in response to confirmatory than disconfirmatory evidence.

\paragraph{Model specification.}
We fit models of the general form:
\begin{equation}
    |\Delta\text{logit}|_{mqi} = \beta_0 + \beta_1 \cdot x_{qi}^{(z)} + \beta_2 \cdot \text{Direction}_i + \gamma_m + u_q + \varepsilon_{mqi}
\end{equation}
where $x_{qi}^{(z)}$ is the topological predictor $z$-scored within each question $q$ (to normalize for graph size), $\text{Direction}_i$ is a binary indicator for strengthen vs.\ negate probes, $\gamma_m$ is a model fixed effect, $u_q \sim \mathcal{N}(0, \sigma_q^2)$ is a random intercept by question, and $\varepsilon_{mqi}$ is residual error. The key parameter is $\beta_1$: a positive $\beta_1$ indicates that more structurally important elements elicit larger probability revisions after controlling for probe direction and model-level baselines.

We fit two variants for each predictor: (1) a \textbf{shared-slope} model (single $\beta_1$ across all models, used for headline results) and (2) an \textbf{interaction model} ($\beta_1 \times \text{model}$, yielding model-specific slopes). For each model, we compute the total slope ($\beta_1 + \beta_{1 \times m}$) and its standard error from the variance--covariance matrix, testing against zero to determine whether that model individually exhibits the topological effect.

\paragraph{Topological predictors.}
We test two node-level predictors, each in a separate LME model:

\begin{itemize}
    \item \textbf{Node betweenness centrality} ($C_B$): fraction of all shortest paths passing through the node. This is a generic graph centrality measure---high-betweenness nodes mediate many paths between pairs of nodes in the network.
    \item \textbf{Outcome mediation}: fraction of factor$\rightarrow$outcome shortest paths passing through the node. Unlike betweenness (which counts paths between all pairs), outcome mediation is outcome-directed and captures how central the node is specifically to the forecasting target.
\end{itemize}

\paragraph{Probe taxonomy.}
Probes are classified into four intervention categories: \textbf{Strengthen} (reinforce existing causal claims), \textbf{Negate} (challenge existing claims), \textbf{Structural Challenge} (challenge the graph architecture---e.g., propose missing edges, reverse edge direction, suggest omitted nodes), and \textbf{Control} (irrelevant information). Within each category, probes target either node-level or edge-level elements, enabling factorial comparison.

\paragraph{Coherence checks.}
We assess reasoning coherence via three supplementary analyses: (1) \textbf{stated-impact ratings}: a GPT-4o-mini judge rates each probe response's stated impact on a 1--5 scale, which we correlate with actual $|\Delta|$; (2) \textbf{Bayesian coherence}: log-odds shift as a function of initial probability; (3) \textbf{embedding separation}: within-question cosine similarity of reasoning embeddings (\texttt{text-embedding-3-large}) for targeted probes vs.\ irrelevant probes.

\paragraph{Cross-model DAG agreement.}
Because models assign different names to semantically equivalent factors (e.g., \texttt{economic\_growth} vs.\ \texttt{economic\_conditions}), exact string matching underestimates structural agreement. We therefore align nodes and edges across models using embedding-based semantic matching before computing similarity.

For each question, we embed every node as ``\texttt{id}: \texttt{description}'' using \texttt{text-embedding-3-large} and construct a cosine-similarity matrix between the two models' node sets. We find the optimal one-to-one alignment via the Hungarian algorithm \cite{kuhn1955hungarian} and count pairs with cosine similarity $\geq 0.7$ as matched. \emph{Semantic node Jaccard} is then $|\text{matched}| \,/\, (|V_1| + |V_2| - |\text{matched}|)$.

For edges, we report two complementary metrics. \emph{Mapped-edge Jaccard} translates graph~1's edges into graph~2's node namespace via the node alignment and computes set intersection---this is conservative, requiring both endpoints to have matched. \emph{Direct-edge Jaccard} instead embeds each edge as ``\texttt{from} $\to$ \texttt{to}: \texttt{mechanism}'' and applies a second Hungarian matching on edge descriptions, capturing cases where models express the same causal relationship through different node decompositions.

For topological comparison, we compute \emph{semantic spectral distance}: both graphs' adjacency matrices are constructed in a unified node namespace (matched nodes share a slot; unmatched nodes receive distinct slots), and we compare the sorted eigenvalues of their normalized Laplacians via $L_2$ distance.

We test all four metrics via permutation tests ($n = 5{,}000$): we shuffle question labels across models and recompute similarity on mismatched question pairs, establishing a null distribution. This tests whether same-question DAGs are more structurally similar than cross-question DAGs---i.e., whether models converge on causal structure for a given forecasting problem.

\section{Experimental Setup}
\label{sec:setup}

\subsection{Models}

We evaluate seven models spanning five architecture families and a range of scales, all accessed via OpenRouter:
\begin{itemize}
    \item \textbf{Llama 3.1 8B Instruct} (8B parameters; released Jul 2024; training cutoff Dec 2023)
    \item \textbf{Llama 3.3 70B Instruct} (70B; released Dec 2024; training cutoff Dec 2023)
    \item \textbf{DeepSeek V3} (685B mixture-of-experts, 37B active; released Mar 2025)
    \item \textbf{Qwen3 235B} (235B MoE, 22B active; released Jul 2025)
    \item \textbf{Qwen3 32B} (32.8B dense; released Apr 2025; training cutoff Mar 2025)
    \item \textbf{Gemini 2.5 Flash Lite} (released Jul 2025; training cutoff Jan 2025)
    \item \textbf{GPT-OSS 120B} (117B MoE; released Aug 2025; training cutoff Jun 2024)
\end{itemize}
Training data cutoffs are reported where officially documented; DeepSeek does not publish a cutoff date. All ForecastBench questions have resolution dates in late 2025, after all known cutoffs. Supplementary analyses (edge permutation, structural ablation, network size, temperature sensitivity, spurious context, propagation decay, and superforecasting comparison) use GPT-OSS 120B as the testbed model.

All models use temperature 0.7 for all stages (causal forecast, probe generation, and probed forecast).

\subsection{Data Collection}

We run the full four-stage pipeline on all 116 questions for each of the seven models, yielding approximately ${\sim}$2,400 probes per model (${\sim}$17,000 total probe responses). Each question--model pair produces an independent causal DAG, set of probes, and probe results.

\subsection{Interactive Explorer}

We release an interactive web application, the \textbf{Causal Forecast Lab}, that allows users to browse the elicited DAGs, probe texts, and probability shifts for every question--model combination. The explorer visualizes the causal network with D3.js force-directed layouts, highlights probe targets by importance tier, and displays shift magnitudes alongside model reasoning. This tool supports qualitative inspection of individual cases and complements the aggregate statistical analyses reported here.

\subsection{Robustness Analyses}

We conduct the following supplementary analyses, all using GPT-OSS 120B as the testbed model on the same 116 high-complexity questions unless otherwise noted:

\paragraph{Test--retest reliability.} We run the full pipeline twice on the same 116 questions for all seven models. For each run we compute the initial probability and the per-question mean $|\Delta_i|$ (averaged across all probes for that question). We then correlate these 116 paired values across runs using Spearman $\rho$: $\rho_p$ for initial probabilities and $\rho_s$ for mean shifts. We also compute semantic node Jaccard similarity between same-question DAGs across runs using the same embedding-based Hungarian matching described in \S\ref{sec:metrics}.

\paragraph{Temperature sensitivity.} We run Stage 1 (causal forecast) at temperatures $T \in \{0.0, 0.3, 0.7, 1.0\}$ on 116 questions using GPT-OSS 120B, with $T = 0.7$ as the pipeline default. For each temperature we compute the mean initial probability with a 95\% bootstrap confidence interval (10,000 resamples). For each pair of temperatures we compute the \textbf{normalized graph edit distance} (nGED) between same-question DAGs, averaged across all 116 questions:
\begin{equation}
    \text{nGED}(G_1, G_2) = \frac{|V_1 \triangle V_2| + |E_1 \triangle E_2|}{|V_1 \cup V_2| + |E_1 \cup E_2|}
\end{equation}
where $\triangle$ denotes symmetric difference. nGED ranges from 0 (identical graphs) to 1 (completely disjoint). This measures the fraction of graph elements (nodes and edges) that must be inserted or deleted to transform one DAG into the other.

\paragraph{Edge permutation condition.} For each of the 116 questions, we construct a permuted DAG by preserving the original nodes but randomly rewiring the edges (maintaining graph density and DAG constraints). We then present this permuted DAG to GPT-OSS 120B as if it were the model's own reasoning and re-run the full probing pipeline. We fit the same shared-slope LME models (betweenness $\rightarrow |\Delta|$ and outcome mediation $\rightarrow |\Delta|$) on the permuted data and compare the grounding coefficients $\beta_1$ to the originals. If structural sensitivity is genuine, $\beta_1$ should degrade when edges no longer support the node's theoretical centrality; if sensitivity is merely sycophantic, $\beta_1$ should be comparable.

\paragraph{Structural ablation.} For each of the 116 questions, we take the GPT-OSS 120B DAG and systematically remove each factor node (along with all its incident edges), then re-elicit a probability forecast with the ablated network. We fit separate LME models predicting $|\Delta|$ from node betweenness and node outcome mediation, with random intercepts by question. This tests whether the model's probability estimate is structurally grounded in the DAG---i.e., whether removing a high-mediation node causes a larger revision than removing a peripheral one.

\paragraph{Spurious context control.} We test whether the DAG elicitation process is robust to irrelevant background information. For each of the 116 questions, we prepend a factual but topically unrelated paragraph (e.g., medieval Scandinavian runestones for a stock price question, deep-sea hydrothermal vent biology for a geopolitics question). Backgrounds are generated by GPT-4o-mini and independently verified as irrelevant by a second GPT-4o-mini call. We then run Stage 1 on the augmented prompts and use a GPT-4o-mini judge to evaluate whether any DAG node was semantically derived from the spurious background. We additionally compare the resulting probabilities and node sets to the original (unaugmented) DAGs.

\paragraph{Network size.} We vary the Stage 1 prompt across two conditions: 6--10 factor nodes (the default) and 12--16 nodes. Both conditions use the same 116 high-complexity questions. The causal complexity filter (see \S\ref{sec:method}) ensures that all questions can naturally sustain 12+ distinct causal factors---without this filter, narrow-domain questions (e.g., weekly temperature comparisons) fail to produce valid large DAGs because the model cannot generate 12+ meaningful, non-redundant factors. We additionally increase the maximum response length from 2,000 to 4,000 tokens for the 12--16 condition, as valid 14--16 node DAGs with edge descriptions require ${\sim}$1,900 tokens on average. For each condition, we fit the shared-slope LME models (betweenness $\rightarrow |\Delta|$ and outcome mediation $\rightarrow |\Delta|$) and compare the grounding coefficients $\beta_1$ across graph sizes. Larger networks provide more intermediate nodes with varying betweenness, potentially enabling finer-grained importance discrimination.

\paragraph{Factor ranking.} To test whether the structural sensitivity revealed by probing reflects explicit or implicit knowledge, we ask each model to rank the key factors for each question by importance---without providing the DAG. We then match the stated ranks to DAG nodes using embedding-based semantic matching (Hungarian algorithm on \texttt{text-embedding-3-large} cosine similarities, threshold $\geq 0.5$) and compute Spearman $\rho$ between stated rank and betweenness centrality. A significant negative correlation would indicate that the model's explicit importance judgments align with DAG topology; a null result would suggest structural sensitivity is implicit (revealed through behavior but not accessible to introspection).

\paragraph{Persuasiveness control.} Because probe generation targets nodes of varying structural importance, probes challenging central nodes may be inherently more persuasive than those challenging peripheral nodes---not due to experimenter framing (which is held constant; see \S\ref{sec:probes}) but because central topics admit stronger arguments. To rule out persuasiveness as a confound, we have an independent LLM judge (GPT-4o-mini) rate the rhetorical persuasiveness of each probe text on a 1--5 scale, blind to the target's structural importance. We then compute partial correlations between structural importance and $|\Delta|$ controlling for persuasiveness, and report mean persuasiveness by importance level. If the importance$\rightarrow$shift relationship survives after controlling for persuasiveness, the topological signal is independent of rhetorical force.

\paragraph{Propagation decay.} Using GPT-OSS 120B, we test whether the model's causal reasoning respects graph locality by measuring how probe effects attenuate with graph distance. For each of the 116 questions, we take the existing node-level probes (\texttt{negate\_high}, \texttt{negate\_low}, \texttt{strengthen}, \texttt{strengthen\_low}) and re-prompt the model with the full DAG and probe text, asking it to rate how each \emph{other} factor node's causal influence would change on a $[-1, +1]$ scale ($-1$ = strongly weakened, $+1$ = strongly reinforced, $0$ = unaffected). We annotate each rated node with its undirected shortest-path distance from the probed node and report (i) mean $|\text{impact}|$ and (ii) fraction of affected nodes (impact $\neq 0$) as a function of graph distance. We test for monotonic decay using Spearman $\rho$ between distance and $|\text{impact}|$ across all node--probe pairs.

\paragraph{Superforecasting prompt comparison.} We run Stage 1 with two system prompts on the same 116 questions: (1) the baseline prompt (used in the main pipeline) and (2) a superforecasting-augmented prompt that explicitly instructs outside-view thinking, granular decomposition, signal--noise discrimination, counterfactual validation, and calibration. We compare the resulting DAGs (node/edge Jaccard similarity, graph density, number of nodes) and probability estimates (Spearman $\rho$, mean absolute difference) to assess whether superforecasting instructions meaningfully alter the elicited causal structure.

\section{System Prompts}
\label{app:prompts}

We provide the full system prompts used at each stage of the pipeline.

\subsection{Stage 1: Causal Forecast}
\label{app:prompt-forecast}

The system prompt instructs the LLM to act as an expert forecaster and produce a probability estimate alongside a causal DAG:

\begin{quote}
\small
\texttt{You are an expert forecaster. Your task is to estimate the probability that a given event will occur, and to represent your reasoning as a directed causal graph.}

\texttt{Build a causal network where:}
\begin{itemize}
\small
\item \texttt{Nodes are factors that influence the outcome}
\item \texttt{Directed edges represent causal mechanisms (A -> B means A causally influences B)}
\item \texttt{One node is the ``outcome'' node representing the forecasted event}
\item \texttt{All other nodes are ``factor'' nodes representing causal drivers}
\end{itemize}

\texttt{You must be well-calibrated: assign probabilities that genuinely reflect your uncertainty. Avoid defaulting to 50\% unless you truly have no information.}

\texttt{Respond with ONLY valid JSON. No other text.}
\end{quote}

The user prompt specifies the question, required JSON schema (probability, nodes with IDs/descriptions/roles, edges with from/to/mechanism, reasoning), and structural constraints:
\begin{itemize}
\small
\item 6--10 factor nodes plus exactly 1 outcome node
\item Each factor must have at least 1 outgoing edge
\item The outcome must have at least 1 incoming edge
\item All edge endpoints must reference valid node IDs
\item Indirect causal chains are required (not just direct links to outcome)
\item Node IDs use \texttt{short\_snake\_case} format
\end{itemize}

\subsection{Stage 2: Probe Generation}
\label{app:prompt-probe-gen}

The system prompt for probe generation:

\begin{quote}
\small
\texttt{You are an expert at constructing probes to test how a forecaster updates their beliefs when elements of their causal reasoning are challenged. You will be given a causal network (nodes and edges) and asked to create a targeted probe that challenges a specific structural element of that network.}

\texttt{The probe should be plausible, specific, and designed to test whether the forecaster can appropriately update their beliefs when confronted with new information about their causal model.}

\texttt{Respond with ONLY valid JSON. No other text.}
\end{quote}

The user prompt provides the question, initial probability, full causal network, and type-specific instructions. Table~\ref{tab:probe_instructions} summarizes the instructions for each probe type.

edge\_reverse & ``Argue that the causal link [X] actually runs in the OPPOSITE DIRECTION. Provide a specific mechanism for reverse causation.'' \\
edge\_structural & ``The network does NOT include a direct link [X]. Argue that this missing link DOES exist and is important. Describe a specific causal mechanism.'' \\

\midrule
\multicolumn{2}{l}{\textit{Control probes}} \\

\end{tabular}
\caption{Probe generation instructions by type. Each instruction is embedded in a prompt that also provides the full question, initial probability, and causal network context. [X] denotes the target node or edge ID and description. Crucially, instructions do not reveal the structural importance of the target to the probe-generating model. \subsection{Stage 3: Probed Forecast}
\label{app:prompt-probed}

The system prompt for the probed forecast stage:

\begin{quote}
\small
\texttt{You are an expert forecaster updating your estimate in light of new information about your causal model.}

\texttt{When presented with new information, consider how it affects your causal network and update your probability estimate accordingly.}

\texttt{Respond with ONLY valid JSON. No other text.}
\end{quote}

The user prompt provides the original question, initial probability $p_0$, the full causal network (nodes and edges with mechanisms), and the probe text. The model outputs \texttt{updated\_probability}, \texttt{shift\_direction}, and \texttt{reasoning}. The prompt does not indicate which elements are structurally important or instruct the model to scale updates by importance---any differential sensitivity to structural importance must arise from the model's own assessment of the DAG.

\subsection{Propagation Decay Prompt}
\label{app:prompt-propagation}

The system prompt for the propagation decay analysis:

\begin{quote}
\small
\texttt{You are an expert forecaster reasoning about how new information propagates through a causal network.}

\texttt{When presented with a challenge to one element of your causal network, consider how the effect ripples through to OTHER factors in the network---not just the final outcome. Some factors may be directly downstream of the challenged element; others may be unaffected.}

\texttt{Rate the impact on each factor using a scale from $-$1.0 (strongly weakened) to +1.0 (strongly reinforced), with 0.0 meaning no effect.}

\texttt{Respond with ONLY valid JSON. No other text.}
\end{quote}

The user prompt provides the original question, initial probability, the full causal network, and the probe text from the main pipeline. The model is asked to rate each \emph{other} factor node's impact and provide one-sentence reasoning per node, plus an updated probability. The prompt instructs: ``factors directly downstream of [target] in the network should be affected more than distant or unrelated factors.''

\section{Probe Target Selection Algorithm}
\label{app:target-selection}

The probe target selection algorithm operates on the output of network analysis and allocates targets as follows:

\begin{enumerate}
\item \textbf{Sort factor nodes} by betweenness centrality (descending), excluding the outcome node.
\item \textbf{Node targets:}
    \begin{itemize}
    \item Top 2 nodes $\rightarrow$ \texttt{negate\_high} (2) + \texttt{strengthen} (2)
    \item Median node $\rightarrow$ \texttt{negate\_medium} (1) + \texttt{strengthen\_medium} (1)
    \item Bottom node $\rightarrow$ \texttt{negate\_low} (1) + \texttt{strengthen\_low} (1)
    \end{itemize}
\item \textbf{Partition edges} into shortest-path (on any shortest path from a factor to the outcome) and peripheral.
\item \textbf{Edge targets:}
    \begin{itemize}
    \item Top 2 shortest-path edges by edge betweenness $\rightarrow$ \texttt{negate\_critical} (2) + \texttt{strengthen\_critical} (2)
    \item Top peripheral edge $\rightarrow$ \texttt{negate\_peripheral} (1) + \texttt{strengthen\_peripheral} (1)
    \item Top shortest-path edge $\rightarrow$ \texttt{reverse} (1)
    \end{itemize}
\item \textbf{Spurious edges:} Select up to 2 non-adjacent node pairs (no existing edge) for \texttt{edge\_structural} probes.
\item \textbf{Control probes:} 2 \texttt{missing\_node} + 2 \texttt{irrelevant} (generated freely by the LLM without a specific target).
\end{enumerate}

The exact count varies by graph structure (19--21 probes, mean ${\sim}$21) because some graphs have fewer shortest-path edges or non-adjacent node pairs.

\section{Metric Definitions and Statistical Tests}
\label{app:metrics}

\paragraph{Bootstrap confidence intervals.} Descriptive metrics (mean shifts by category) are reported with 95\% bootstrap confidence intervals using 10,000 resamples. LME models report Wald confidence intervals and Satterthwaite $p$-values.

\paragraph{Test--retest reliability.} We report Spearman $\rho$ between paired measurements across runs (initial probabilities $\rho_p$, per-question mean shifts $\rho_s$) and semantic node Jaccard similarity for graph structure. Per-model results: Llama-8B ($\rho_p = .809$, $\rho_s = .605$), Llama-70B ($\rho_p = .866$, $\rho_s = .749$), DeepSeek-V3 ($\rho_p = .703$, $\rho_s = .705$), Qwen3-235B ($\rho_p = .960$, $\rho_s = .871$), Gemini-Flash-Lite ($\rho_p = .877$, $\rho_s = .564$). All $n = 100$ questions per model.

\section{Additional Analysis Details}
\label{app:additional}

\subsection{Spurious Context Control}

We test whether prepending irrelevant background information contaminates DAG construction. For each of the 116 questions, we pair the original question with a factual paragraph drawn from an unrelated domain (e.g., the biomechanics of woodpecker skull shock absorption, competitive pigeon racing in Belgium, the archaeology of Nabataean water management at Petra). Topics are drawn from a pool of obscure domains and assigned to questions such that no topical overlap exists. Each background is generated by GPT-4o-mini conditioned on the assigned topic and verified as irrelevant by a separate GPT-4o-mini call; any flagged pair is regenerated with a different topic.

We run Stage 1 (causal forecast) on the augmented prompts using GPT-OSS 120B and evaluate incorporation with a GPT-4o-mini judge that examines each DAG node for semantic relatedness to the spurious background.

\paragraph{Results.} %TODO: Update numbers once GPT-OSS spurious context run completes.
Across all 116 questions, the judge found \textbf{zero} DAG nodes derived from the spurious background (0/116 contamination rate). The model consistently constructed topically appropriate causal networks despite the irrelevant preamble. These results confirm that the DAG elicitation process is robust to irrelevant context injection.

\subsection{Superforecasting Prompt Comparison}

We compare DAGs produced by two Stage 1 system prompts on the same 116 questions using GPT-OSS 120B:

\paragraph{Baseline prompt.} The standard prompt used in the main pipeline, which instructs the LLM to ``be well-calibrated'' and produce a causal network (Appendix~\ref{app:prompt-forecast}).

\paragraph{Superforecasting-augmented prompt.} Adds five explicit principles before the network construction instructions:
\begin{enumerate}
\small
\item \textit{Outside view first:} ``Consider the base rate. How often do events like this happen historically? Start from reference classes before incorporating case-specific details.''
\item \textit{Granular decomposition:} ``Break the problem into independent sub-components. Each node should represent a distinct, measurable factor.''
\item \textit{Signal--noise discrimination:} ``Only include factors that have genuine causal relevance. Resist the temptation to add factors just because they are topically related.''
\item \textit{Counterfactual validation:} ``For each causal link, ask: `If this factor were different, would the outcome probability actually change?' ''
\item \textit{Calibration:} ``Assign probabilities that genuinely reflect your uncertainty. Use historical base rates as anchors.''
\end{enumerate}

We compare the two conditions on: probability Spearman $\rho$ ($= 0.845$), mean absolute probability difference ($0.061$), semantic node Jaccard ($0.423$), semantic mapped-edge Jaccard ($0.113$), aligned spectral distance ($0.475$), mean node count (baseline $6.8$, SF $6.8$; Wilcoxon $p = 0.239$), mean edge count ($6.9$ vs $6.8$; $p = 0.518$), and graph density ($0.172$ vs $0.175$; $p = 0.214$). None of the structural differences are significant, and probability estimates are highly correlated, indicating that explicit superforecasting instructions do not produce qualitatively different causal models---the LLM's default reasoning already incorporates similar structure.

\subsection{Question Difficulty Rating}
\label{app:difficulty}

A question's initial probability $p_0$ reflects how likely an event is, but not how \emph{hard} it is to forecast accurately.
A 50\% estimate may reflect well-understood equipoise or deep epistemic opacity; a 95\% estimate may be trivially easy or hinge on opaque insider dynamics.
To disentangle forecasting difficulty from the base rate, we construct an independent difficulty ranking using pairwise LLM-as-judge comparison.
Conceptually, this ranking captures a question's position on the aleatoric--epistemic spectrum: easy questions tend to involve well-understood stochastic processes (weather, economic indicators) with primarily aleatoric uncertainty, while hard questions involve novel situations with opaque causal mechanisms (geopolitical contingencies, long-horizon technology forecasts) where epistemic uncertainty dominates.

\paragraph{Tournament design.}
We run a Swiss-system tournament over the 116 questions using GPT-4o-mini as judge.
In each round, questions with similar current ratings are paired; the judge sees both question texts and selects which is harder to predict accurately, considering information availability, causal complexity, temporal sensitivity, domain opacity, reference class ambiguity, and strategic interaction.
The judge is explicitly instructed not to equate proximity to 50\% with difficulty.
Presentation order is randomized within each pair to control for position bias.
After 15 rounds (745 total comparisons), we compute Elo ratings from the win--loss record using a $K$-factor of 32.

\paragraph{Linear mixed-effects model.}
We test whether Elo difficulty predicts belief sensitivity metrics beyond what $|p_0 - 0.5|$ already explains.
For each (model, question) pair, we compute per-question values of mean absolute shift, asymmetry index, and mean judge ratings (reasoning coherence and expressed uncertainty, each averaged across all probe ratings for that question).
We fit linear mixed-effects models of the form:
\begin{equation}
    y_{mq} = \beta_0 + \beta_1 \cdot \text{Elo}_q^{(z)} + \beta_2 \cdot |p_0 - 0.5|_q^{(z)} + u_m + \varepsilon_{mq}
\end{equation}
where $y_{mq}$ is the metric for model $m$ on question $q$, predictors are $z$-scored, $u_m \sim \mathcal{N}(0, \sigma_m^2)$ is a random intercept for model, and $\varepsilon_{mq}$ is residual error.
We assess the incremental contribution of Elo via likelihood ratio tests comparing the full model to a $|p_0 - 0.5|$-only baseline, and report marginal and conditional $R^2$ \cite{nakagawa2013general}.
As a validation check, we test whether Elo ratings correlate with inter-model disagreement (SD of initial probabilities across the five models for the same question), controlling for $|p_0 - 0.5|$.
